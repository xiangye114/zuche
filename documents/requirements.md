## 1. 引言

### 1.1 编写目的
本文档详细描述了租车管理系统订单状态管理流程优化的功能需求，为开发团队提供明确的开发指导，为测试团队提供测试依据。

### 1.2 项目背景
现有租车管理系统的订单状态管理功能较为简单，无法满足复杂的业务流程需求。本次优化旨在完善订单生命周期管理，实现完整的6状态订单流程。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 订单状态 | 订单在生命周期中所处的阶段 |
| 状态流转 | 订单从一个状态转换到另一个状态的过程 |
| 车辆检查 | 取车或还车时对车辆状况的检查 |
| 结算 | 订单完成后进行费用清算的过程 |

### 1.4 订单状态定义

| 状态值 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 待确认 | 订单创建后等待管理员确认 |
| 1 | 已确认 | 管理员确认通过，等待取车 |
| 2 | 已取车 | 用户已取车，用车中 |
| 3 | 已还车 | 用户已还车，待结算 |
| 4 | 已完成 | 订单结算完成，流程结束 |
| 5 | 已取消 | 订单已取消，流程结束 |

## 2. 功能需求

### 2.1 待确认状态功能

#### 2.1.1 功能描述
用户成功提交订单后，订单自动进入"待确认"状态。管理员需要在此状态下完成订单审核工作。

#### 2.1.2 功能清单

**FR-001: 订单信息查看**
- 管理员可查看完整订单信息
- 显示字段：订单号、用户信息、租车时间、车型信息、取还车地点、订单金额

**FR-002: 车辆可用性检查**
- 系统自动检查所选时间段内车辆是否空闲
- 如车辆已被占用，系统提示冲突信息
- 检查逻辑：查询该车辆在该时间段内是否存在状态不为"已取消"和"已完成"的订单

**FR-003: 用户资质审核**
- 显示用户身份信息（姓名、身份证号、驾驶证信息）
- 显示用户历史订单记录
- 显示用户信用评分（如有）
- 标记异常用户（如历史违约记录）

**FR-004: 确认订单操作**
- 管理员可执行"确认通过"操作
- 操作后订单状态变为"已确认"
- 记录操作人、操作时间、备注信息
- 发送通知给用户（短信/邮件）

**FR-005: 拒绝订单操作**
- 管理员可执行"确认不通过"操作
- 必须选择拒绝原因（标准原因或自定义）
- 可选操作：退回修改或直接取消
- 如选择取消，订单状态变为"已取消"
- 记录操作人、操作时间、拒绝原因、备注信息
- 发送通知给用户

**FR-006: 拒绝原因管理**
- 系统提供标准拒绝原因列表
- 支持自定义拒绝原因
- 拒绝原因分类：车辆问题、用户资质问题、时间冲突、其他

### 2.2 已确认状态功能

#### 2.2.1 功能描述
订单通过管理员确认后进入"已确认"状态。此阶段主要进行车辆准备工作。

#### 2.2.2 功能清单

**FR-007: 车辆准备工作清单**
- 显示车辆准备检查项列表
- 检查项包括：车辆清洁、油量检查、轮胎检查、设备检查等
- 支持逐项确认
- 显示准备完成进度

**FR-008: 车辆检查记录**
- 支持录入取车前车辆检查信息
- 检查项目：外观、内饰、油量、里程、设备功能
- 支持上传检查照片
- 记录检查人、检查时间
- 生成检查报告

**FR-009: 取车提醒功能**
- 显示用户预计取车时间
- 取车前30分钟自动提醒管理员
- 支持手动发送取车提醒给用户

**FR-010: 取车手续办理**
- 记录用户实际取车时间
- 核对用户身份信息
- 签署取车确认单
- 记录取车经办人

**FR-011: 确认取车操作**
- 管理员执行"确认取车"操作
- 订单状态变为"已取车"
- 记录取车时间、经办人、备注
- 发送通知给用户

### 2.3 已取车状态功能

#### 2.3.1 功能描述
用户成功取车后，订单进入"已取车"状态。此阶段主要监控车辆使用情况。

#### 2.3.2 功能清单

**FR-012: 车辆使用监控**
- 显示当前车辆状态（使用中）
- 如配备GPS，显示车辆实时位置
- 显示预计还车时间
- 显示已使用时长

**FR-013: 取车信息展示**
- 显示取车门店信息
- 显示实际取车时间
- 显示取车经办人
- 显示取车时的车辆状况（里程、油量）

**FR-014: 异常上报功能**
- 用户可上报车辆使用中的异常情况
- 异常类型：故障、事故、违章等
- 支持上传照片和描述
- 管理员收到异常通知

**FR-015: 还车提醒功能**
- 还车前2小时自动提醒用户
- 显示还车地点和注意事项
- 支持手动发送还车提醒

**FR-016: 确认还车操作**
- 用户归还车辆后，管理员执行"确认还车"操作
- 订单状态变为"已还车"
- 记录还车时间、经办人、备注
- 触发还车检查流程

### 2.4 已还车状态功能

#### 2.4.1 功能描述
用户归还车辆后，订单进入"已还车"状态。此阶段主要进行车辆检查和费用结算。

#### 2.4.2 功能清单

**FR-017: 还车检查功能**
- 支持录入还车时车辆检查信息
- 检查项目与取车时一致
- 自动对比取车和还车时的差异
- 突出显示异常项（如新增损伤、油量差异）
- 支持上传检查照片

**FR-018: 损伤记录功能**
- 如发现车辆损伤，记录损伤详情
- 损伤类型：划痕、凹陷、玻璃破损等
- 记录损伤位置、程度、照片
- 评估维修费用

**FR-019: 费用结算计算**
- 基础租金计算
- 超时费用计算（如适用）
- 超里程费用计算（如适用）
- 油费差额计算（如适用）
- 损伤赔偿费用
- 优惠折扣计算
- 押金退还计算
- 生成费用明细清单

**FR-020: 结算确认操作**
- 管理员确认结算金额
- 支持调整结算金额（需填写调整原因）
- 用户确认结算（可选）
- 订单状态变为"已完成"
- 记录结算人、结算时间
- 触发退款流程（如适用）

### 2.5 已完成状态功能

#### 2.5.1 功能描述
订单结算完成后进入"已完成"状态。此阶段主要进行订单归档和数据分析。

#### 2.5.2 功能清单

**FR-021: 订单归档功能**
- 订单完整信息存档
- 包括所有操作记录、检查记录、费用明细
- 支持订单信息导出（PDF/Excel）

**FR-022: 数据统计接口**
- 营收统计分析
- 车辆使用频率分析
- 订单状态分布统计
- 取消原因统计分析
- 支持按时间范围、门店、车型筛选

**FR-023: 订单查询导出**
- 支持多条件组合查询
- 支持批量导出订单数据
- 支持自定义导出字段

### 2.6 已取消状态功能

#### 2.6.1 功能描述
订单在"待确认"或"已确认"状态下可被取消。

#### 2.6.2 功能清单

**FR-024: 取消记录功能**
- 记录取消时间
- 记录取消人（用户/管理员/系统）
- 记录取消原因（标准原因或自定义）
- 记录取消备注

**FR-025: 取消原因统计**
- 按取消原因分类统计
- 按取消人分类统计
- 按时间范围统计
- 可视化图表展示

**FR-026: 订单锁定功能**
- 取消后订单锁定
- 不允许任何后续操作
- 仅允许查看订单信息

**FR-027: 退款流程关联**
- 如已支付，触发退款流程
- 记录退款金额、退款时间
- 支持部分退款

### 3.2 接口详细说明

#### 3.2.1 确认订单接口

**请求URL**: `/admin/orders/{id}/confirm`

**请求方式**: PUT

**请求参数**:
```json
{
  "operatorName": "操作人姓名",
  "remark": "备注信息"
}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "确认订单成功",
  "data": null
}
```

#### 3.2.2 拒绝订单接口

**请求URL**: `/admin/orders/{id}/reject`

**请求方式**: PUT

**请求参数**:
```json
{
  "operatorName": "操作人姓名",
  "reasonCode": "拒绝原因编码",
  "remark": "备注信息"
}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "拒绝订单成功",
  "data": null
}
```

#### 3.2.3 提交车辆检查接口

**请求URL**: `/admin/orders/{id}/car-checks`

**请求方式**: POST

**请求参数**:
```json
{
  "checkType": 1,
  "checkItems": [
    {
      "checkItem": "外观检查",
      "checkResult": 1,
      "description": "",
      "photos": "[\"url1\", \"url2\"]"

```

**响应示例**:
```json
{
  "code": 200,
  "message": "提交检查记录成功",
  "data": null
}
```

#### 3.2.4 结算订单接口

**请求URL**: `/admin/orders/{id}/settle`

**请求方式**: PUT

**请求参数**:
```json
{
  "settlementAmount": 1500.00,
  "operatorName": "结算人",
  "remark": "结算备注"
}
```

**响应示例**:
```json
{
  "code": 200,
  "message": "结算成功",
  "data": null
}
```

### 4.3 数据迁移脚本

```sql
-- 更新租车天数
UPDATE orders 
SET rental_days = DATEDIFF(end_time, start_time) 
WHERE rental_days IS NULL AND start_time IS NOT NULL AND end_time IS NOT NULL;

-- 初始化费用字段
UPDATE orders 
SET rental_fee = total_price,
    insurance_fee = 0,
    service_fee = 0,
    overtime_fee = 0,
    mileage_fee = 0,
    discount_amount = 0
WHERE rental_fee IS NULL;
```

#### 取车检查项目

5. 里程表记录
6. 灯光检查

3. 轮胎检查
4. 油量检查
5. 里程表记录
6. 新增损伤记录
