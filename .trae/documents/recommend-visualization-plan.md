
# 推荐模块可视化展示方案

## 一、当前推荐模块分析

### 1.1 后端推荐服务能力

| 方法 | 功能描述 | 推荐算法 |
|------|----------|----------|
| `getHotCars()` | 获取热门车型 | 按创建时间降序 |
| `getRecommendForUser()` | 用户个性化推荐 | 基于历史订单+收藏记录，计算偏好分数 |
| `getSimilarCars()` | 相似车型推荐 | 基于品牌/类别匹配 |

### 1.2 当前前端展示现状

- **首页**：仅展示"热门推荐"区域，使用CarCard组件展示4辆车
- **问题**：未区分个性化推荐、热门推荐、相似推荐，缺少推荐理由可视化

### 1.3 推荐算法评分机制

`getRecommendForUser()` 的评分规则：
- **品牌匹配**：+10分
- **类别匹配**：+5分
- **价格相近（<50元）**：+8分
- **价格相近（50-100元）**：+5分

## 三、技术实现方案

### 3.1 后端改造

**步骤1：修改Service层，返回推荐评分信息**

创建 `CarRecommendVO.java`，扩展Car实体，增加推荐相关字段：

```java
public class CarRecommendVO extends Car {
    private Integer recommendScore;      // 推荐分数
    private List<String> recommendReasons; // 推荐理由列表
    // getter/setter
}
```

**步骤2：修改RecommendServiceImpl**

修改 `getRecommendForUser()` 方法，返回带评分信息的VO对象。

### 3.2 前端改造

**步骤1：修改CarCard组件**

增加推荐理由展示区域，支持显示：
- 推荐分数进度条
- 推荐理由标签（品牌匹配、价格合适、热门车型等）

**步骤2：修改Home.vue**

- 增加"为你推荐"区域（登录用户）
- 保留"热门推荐"区域（访客/用户）
- 调整布局为两列卡片网格

**步骤3：修改CarDetail.vue**

在详情页底部增加"相似车型推荐"横向滚动区域。

### 3.3 推荐理由枚举定义

| 理由代码 | 显示文本 | 触发条件 |
|----------|----------|----------|
| `BRAND_MATCH` | 品牌匹配 | 用户历史包含同品牌车辆 |
| `CATEGORY_MATCH` | 类别匹配 | 用户历史包含同类别车辆 |
| `PRICE_MATCH` | 价格合适 | 价格接近用户历史平均 |
| `HOT` | 热门车型 | 近期新增/订单量高 |
| `SIMILAR` | 相似推荐 | 同品牌/同类别的其他车型 |

## 六、预期效果

### 6.1 用户视角

1. **首页**：登录用户看到"为你推荐"和"热门车型"两个区域
2. **卡片**：每辆车显示推荐理由标签，如"品牌匹配"、"价格合适"
3. **详情页**：底部展示相似车型横向滚动列表

### 6.2 业务价值

- **提升转化率**：用户理解推荐原因，更容易点击
- **增加透明度**：推荐算法不再是黑盒
- **个性化体验**：增强用户感知的定制化服务

---

## 七、风险与注意事项

1. **性能影响**：推荐计算增加数据库查询，需评估响应时间
2. **未登录用户**：降级为仅展示热门推荐
3. **空数据处理**：无推荐结果时显示热门车型作为兜底
4. **样式兼容性**：确保移动端响应式布局正常
