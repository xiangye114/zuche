# 在线租车网站需求分析计划

## 一、项目概述

### 1.1 项目背景
随着共享经济的发展和人们出行需求的多样化，在线租车服务已成为一种重要的出行方式。本项目旨在设计并实现一个功能完善、用户体验良好的在线租车网站。

### 1.2 项目目标
- 为用户提供便捷的在线租车服务
- 为管理员提供高效的车辆和订单管理平台
- 实现完整的租车业务流程

### 1.3 技术栈
| 层级 | 技术 |
|------|------|
| 前端框架 | Vue.js 3.x |
| UI组件库 | Element Plus |
| 后端框架 | Java + Spring Boot |
| 数据库 | MySQL |
| 缓存 | Redis（可选） |
| 身份认证 | JWT |

---

## 二、用户角色分析

### 2.1 普通用户（租车客户）
- 浏览车辆信息
- 在线预订车辆
- 管理个人订单
- 评价租车体验

### 2.2 系统管理员
- 管理车辆信息
- 处理用户订单
- 管理用户账户
- 查看数据统计
- 管理门店信息

### 2.3 门店管理员
- 管理所在门店的车辆信息（上架、下架、编辑）
- 处理所在门店的订单（确认取车、确认还车）
- 查看所在门店的订单统计
- 管理门店车辆库存状态

#### 3.1.2 首页租车流程模块（重构）
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 时间地点选择 | 首页提供取车时间、还车时间、取车门店、还车门店选择 | 高 |

4. 点击"立即租车"，系统根据条件筛选可用车辆

**确认取车流程**：
1. 用户到店出示订单信息
2. 门店管理员扫描或输入订单号
3. 核对用户身份和订单信息
4. 确认车辆状况良好
5. 点击"确认取车"，填写备注信息
6. 订单状态变更为"用车中"

**确认还车流程**：
1. 用户到店还车
2. 门店管理员检查车辆状况
3. 核对还车时间和费用
4. 点击"确认还车"，填写备注信息
5. 订单状态变更为"已还车"

## 四、非功能性需求

### 4.1 性能需求
- 页面加载时间 < 3秒
- 数据库查询响应时间 < 1秒

### 4.2 安全需求
- 用户密码加密存储（BCrypt）
- 使用JWT进行身份认证
- 防止SQL注入、XSS攻击
- 用户敏感操作日志记录

### 4.3 兼容性需求
- 支持主流浏览器（Chrome、Firefox、Edge）
- PC端Web网站

## 十、智能AI客服模块详细设计

### 10.1 功能概述
智能AI客服模块为用户提供7x24小时在线咨询服务，能够自动回答用户关于租车流程、车辆信息、订单状态等常见问题，提升用户体验，降低人工客服成本。

### 10.2 技术实现方案

#### 方案一：调用第三方AI API（推荐）
使用成熟的AI大模型API，如：
- **百度文心一言 API**
- **阿里通义千问 API**
- **智谱AI（GLM）API**
- **OpenAI API**

**优点**：
- 开发成本低，无需训练模型
- 智能程度高，理解能力强
- 维护简单，按量付费

**缺点**：
- 需要联网调用
- 有一定的API调用费用

#### 方案二：本地部署开源模型
使用开源大模型本地部署：
- **ChatGLM**（清华开源）
- **Llama**
- **Qwen**

**优点**：
- 数据安全，不外泄
- 无API调用费用

**缺点**：
- 需要高性能服务器
- 部署和维护复杂

#### 方案三：基于规则的智能客服（简单方案）
使用关键词匹配 + 预设问答库：
- 建立常见问题知识库
- 关键词匹配返回答案
- 支持多轮对话引导

**优点**：
- 实现简单，无需AI模型
- 完全可控

**缺点**：
- 智能程度有限
- 需要维护大量问答数据

### 10.3 推荐方案：第三方AI API + 知识库增强

```java
@Service
public class ChatServiceImpl implements ChatService {
    
    @Value("${ai.api.key}")
    private String apiKey;
    
    @Value("${ai.api.url}")
    private String apiUrl;
    
    public ChatMessageVO chat(Long sessionId, String userMessage) {
        // 1. 保存用户消息
        ChatMessage userMsg = saveUserMessage(sessionId, userMessage);
        
        // 2. 检索知识库
        String knowledge = searchKnowledge(userMessage);
        
        // 3. 构建提示词
        String systemPrompt = buildSystemPrompt(knowledge);
        
        // 4. 调用AI API
        String aiResponse = callAiApi(systemPrompt, userMessage, getSessionHistory(sessionId));
        
        // 5. 保存AI回复
        ChatMessage aiMsg = saveAiMessage(sessionId, aiResponse);
        
        return convertToVO(aiMsg);
    }
    
    private String buildSystemPrompt(String knowledge) {
        return "你是一个专业的租车网站客服助手。请根据以下知识库内容回答用户问题：\n" 
            + knowledge + "\n\n请用友好、专业的语气回答，如果问题超出知识库范围，请引导用户联系人工客服。";
    }
}
```

## 十一、个性化推荐模块详细设计

### 11.1 功能概述
个性化推荐模块通过分析用户的历史订单、浏览记录、收藏行为等数据，利用协同过滤算法为用户推荐合适的车辆类型和租车套餐，提升用户选择效率和满意度。

### 11.2 推荐算法设计

#### 11.2.1 协同过滤算法原理
协同过滤是推荐系统中最经典的算法，分为两种：

| 算法类型 | 原理 | 适用场景 |
|---------|------|---------|
| 基于用户(User-Based) | 找到与当前用户相似的其他用户，推荐他们喜欢的车辆 | 用户量较大时 |
| 基于物品(Item-Based) | 找到与用户已租/浏览车辆相似的其他车辆进行推荐 | 车辆量较大时（推荐） |

**本项目推荐使用基于物品的协同过滤算法**，原因：
- 车辆数量相对稳定，相似度矩阵可预先计算
- 计算效率高，实时性好
- 推荐结果可解释性强

### 11.4 核心算法实现

#### 11.4.1 相似度计算（余弦相似度）

```java
@Service
public class RecommendService {
    
    public double calculateSimilarity(Long carIdA, Long carIdB) {
        List<UserBehavior> behaviorsA = userBehaviorRepository.findByCarId(carIdA);
        List<UserBehavior> behaviorsB = userBehaviorRepository.findByCarId(carIdB);
        
        Map<Long, Double> vectorA = buildBehaviorVector(behaviorsA);
        Map<Long, Double> vectorB = buildBehaviorVector(behaviorsB);
        
        double dotProduct = 0.0;
        double normA = 0.0;
        double normB = 0.0;
        
        Set<Long> allUsers = new HashSet<>();
        allUsers.addAll(vectorA.keySet());
        allUsers.addAll(vectorB.keySet());
        
        for (Long userId : allUsers) {
            double scoreA = vectorA.getOrDefault(userId, 0.0);
            double scoreB = vectorB.getOrDefault(userId, 0.0);
            dotProduct += scoreA * scoreB;
            normA += scoreA * scoreA;
            normB += scoreB * scoreB;
        }
        
        if (normA == 0 || normB == 0) return 0.0;
        return dotProduct / (Math.sqrt(normA) * Math.sqrt(normB));
    }
    
    private Map<Long, Double> buildBehaviorVector(List<UserBehavior> behaviors) {
        Map<Long, Double> vector = new HashMap<>();
        for (UserBehavior behavior : behaviors) {
            vector.merge(behavior.getUserId(), behavior.getScore(), Double::sum);
        }
        return vector;
    }
}
```

```java
public List<CarRecommendVO> getRecommendations(Long userId, int limit) {
    List<UserBehavior> userBehaviors = userBehaviorRepository.findByUserId(userId);
    
    if (userBehaviors.isEmpty()) {
        return getHotRecommendations(limit);
    }
    
    Map<Long, Double> userPreferences = buildUserPreferences(userBehaviors);
    
    Map<Long, Double> candidateScores = new HashMap<>();
    
    for (UserBehavior behavior : userBehaviors) {
        List<CarSimilarity> similarCars = similarityRepository
            .findByCarIdAOrderBySimilarityDesc(behavior.getCarId());
        
        for (CarSimilarity sim : similarCars) {
            Long candidateId = sim.getCarIdB();
            if (hasRented(userId, candidateId)) continue;
            
            double score = userPreferences.get(behavior.getCarId()) 
                         * sim.getSimilarity();
            candidateScores.merge(candidateId, score, Double::sum);
        }
    }
    
    return candidateScores.entrySet().stream()
        .sorted(Map.Entry.<Long, Double>comparingByValue().reversed())
        .limit(limit)
        .map(entry -> buildRecommendVO(entry.getKey(), entry.getValue()))
        .collect(Collectors.toList());
}
```

```vue
        <el-button type="primary" @click="generateImage" :loading="generating">
          生成
        </el-button>
      </template>
    </el-dialog>
  </div>
</template>

<script setup>
import { ref, onMounted } from 'vue'
import { imageApi } from '@/api/image'

const props = defineProps(['carId'])
const images = ref([])
const showGenerateDialog = ref(false)
const generateAngle = ref('front')
const customPrompt = ref('')
const generating = ref(false)

const generateImage = async () => {
  generating.value = true
  try {
    await imageApi.generate(props.carId, {
      angle: generateAngle.value,
      customPrompt: customPrompt.value
    })
    await loadImages()
    showGenerateDialog.value = false
  } finally {
    generating.value = false
  }
}

const loadImages = async () => {
  const res = await imageApi.getList(props.carId)
  images.value = res.data
}

onMounted(loadImages)
</script>
```
