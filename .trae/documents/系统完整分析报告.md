# 在线租车网站系统 - 完整分析报告

## 一、系统概述

### 1.1 系统名称
**在线租车网站系统**（Online Car Rental System）

### 1.2 系统定位
一个完整的在线租车服务平台，提供车辆浏览、在线预订、订单管理、智能客服等一站式服务。

### 1.3 技术架构
- **架构模式**：前后端分离架构
- **前端技术**：Vue 3 + Vite + Element Plus + Pinia
- **后端技术**：Spring Boot 2.7 + MyBatis Plus + Spring Security
- **数据库**：MySQL 5.7+
- **缓存**：Redis
- **认证方式**：JWT Token

### 2.3 订单管理模块（核心模块）

#### 2.3.1 功能描述
订单管理是本系统的核心模块，实现了完整的6状态订单生命周期管理：

**订单状态流转**：
```
待确认(0) → 已确认(1) → 已取车(2) → 已还车(3) → 已完成(4)
    ↓
已取消(5)
```

**各状态功能**：
1. **待确认**：订单创建后等待管理员确认
2. **已确认**：管理员确认通过，等待取车
3. **已取车**：用户已取车，用车中
4. **已还车**：用户已还车，待结算
5. **已完成**：订单结算完成
6. **已取消**：订单已取消

#### 2.3.2 核心代码实现

**订单实体类 (Order.java)** - 包含40+字段：

**订单服务实现 (OrderServiceImpl.java)** - 核心方法详解：

1. **订单创建 (create)**：
   - 自动生成订单号（ORD + 时间戳 + 随机数）
   - 计算租期天数（使用ChronoUnit.DAYS.between）
   - 计算租金（日租金 × 天数）

2. **确认订单 (confirmOrder)**：
   - 检查订单状态必须为"待确认"
   - 检查车辆可用性（时间段内是否有冲突订单）
   - 更新订单状态为"已确认"
   - 记录操作日志

3. **拒绝订单 (rejectOrder)**：
   - 支持选择拒绝原因
   - 更新订单状态为"已取消"
   - 记录取消原因和备注

4. **确认取车 (confirmPickup)**：
   - 检查订单状态必须为"已确认"
   - 记录实际取车时间、操作人、备注
   - 更新订单状态为"已取车"

5. **确认还车 (confirmReturn)**：
   - 检查订单状态必须为"已取车"
   - 记录实际还车时间、操作人、备注
   - 更新订单状态为"已还车"

6. **订单结算 (settleOrder)**：
   - 检查订单状态必须为"已还车"
   - 计算实际租车天数（基于实际取还车时间）
   - 计算超时费用（分档计费：1-4小时按小时收费，5-24小时按半天收费，超过24小时按天收费）
   - 支持手动调整结算金额
   - 更新订单状态为"已完成"

7. **车辆可用性检查 (checkCarAvailability)**：
   - 查询指定时间段内是否存在冲突订单
   - 排除已取消和已完成的订单
   - 使用复杂的时间范围查询逻辑

**前端订单管理页面 (OrderManage.vue)** - 功能详解：

## 三、系统安全设计

### 3.1 JWT认证机制

**JWT工具类 (JwtUtils.java)**：
```java
@Component
public class JwtUtils {
    @Value("${jwt.secret}")
    private String secret;
    
    @Value("${jwt.expiration}")
    private Long expiration; // 24小时
    
    public String generateToken(Long userId, String username, Integer role) {
        Map<String, Object> claims = new HashMap<>();
        claims.put("userId", userId);
        claims.put("username", username);
        claims.put("role", role);
        
        return Jwts.builder()
                .claims(claims)
                .subject(username)
                .issuedAt(new Date())
                .expiration(new Date(System.currentTimeMillis() + expiration))
                .signWith(getSecretKey())
                .compact();
    }
}
```

### 3.2 Spring Security配置

**安全配置 (SecurityConfig.java)**：
- 禁用CSRF（前后端分离场景）
- 配置JWT过滤器
- 公开接口配置（登录、注册、车辆列表等）
- 需要认证的接口配置

### 3.3 密码加密

使用BCryptPasswordEncoder进行密码加密存储，确保用户密码安全。

系统的核心亮点在于**完整的订单状态管理流程**，实现了从订单创建到结算的6状态流转，支持车辆检查、费用计算、日志记录等功能，能够满足实际租车业务的需求。

系统代码结构清晰，注释完善，易于维护和扩展，适合作为毕业设计项目。
