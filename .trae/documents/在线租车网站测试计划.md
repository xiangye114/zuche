## 一、测试概述

### 1.1 测试目标

本测试计划旨在确保在线租车网站系统的功能完整性、性能稳定性、安全可靠性，验证系统是否满足需求规格说明书中的所有功能需求和非功能需求。

### 1.2 测试原则

- **全面性原则**：覆盖所有功能模块和业务场景
- **独立性原则**：测试活动独立于开发活动
- **可追溯性原则**：测试用例与需求一一对应
- **可重复性原则**：测试用例可重复执行，结果可验证

### 1.3 测试阶段

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  单元测试   │───▶│  集成测试   │───▶│  系统测试   │───▶│  验收测试   │
│  (开发阶段) │    │  (接口联调) │    │  (全量测试) │    │  (用户验收) │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

## 三、测试策略

### 3.1 测试类型策略

#### 3.1.1 功能测试

**测试方法**：黑盒测试
**测试技术**：等价类划分、边界值分析、场景法、判定表
**执行方式**：手工测试 + 自动化测试

#### 3.1.2 接口测试

**测试工具**：Postman / JMeter
**测试内容**：
- 接口功能正确性
- 参数校验（必填、类型、长度、范围）
- 返回结果验证
- 错误码验证
- 性能基准测试

#### 3.1.3 性能测试

**测试工具**：JMeter / LoadRunner
**测试指标**：
| 指标项 | 目标值 | 说明 |
|-------|-------|------|
| 页面加载时间 | < 3秒 | 首屏加载完成 |
| 接口响应时间 | < 1秒 | 95%的请求 |
| 并发用户数 | 100+ | 同时在线用户 |
| 吞吐量 | 50 TPS | 每秒事务数 |
| 错误率 | < 0.1% | 系统错误率 |

#### 3.1.4 安全测试

**测试内容**：
- 身份认证测试（JWT有效性、过期处理）
- 权限控制测试（水平越权、垂直越权）
- 输入验证测试（SQL注入、XSS、CSRF）
- 敏感数据保护（密码加密、传输加密）

### 3.2 测试用例设计方法

| 测试方法 | 适用场景 | 示例 |
|---------|---------|------|
| 等价类划分 | 输入条件有明确范围 | 手机号、价格区间 |
| 边界值分析 | 数值范围测试 | 分页参数、时间范围 |
| 场景法 | 业务流程测试 | 租车流程、订单状态流转 |
| 判定表 | 多条件组合 | 搜索筛选条件 |
| 错误推测法 | 常见错误场景 | 异常参数、网络异常 |

## 八、风险评估

### 8.1 测试风险

| 风险编号 | 风险描述 | 风险等级 | 应对措施 |
|---------|---------|---------|---------|
| R-001 | 需求变更频繁 | 高 | 建立需求变更控制流程，及时更新测试用例 |
| R-002 | 测试环境不稳定 | 中 | 提前搭建环境，定期备份，制定环境恢复方案 |
| R-003 | 测试数据不足 | 中 | 提前准备测试数据，覆盖各种业务场景 |
| R-004 | 开发人员修复缺陷不及时 | 中 | 建立缺陷跟踪机制，每日站会同步 |
| R-005 | AI功能测试结果不稳定 | 高 | 多次测试取平均值，设置合理容错范围 |
| R-006 | 第三方API服务不稳定 | 中 | 准备Mock数据，制定降级方案 |

### 8.2 风险应对策略

| 策略类型 | 具体措施 |
|---------|---------|
| 规避 | 提前识别风险，调整测试计划避免风险发生 |
| 减轻 | 增加测试资源，延长测试时间，降低风险影响 |
| 转移 | 引入自动化测试，将重复工作转移给工具 |
| 接受 | 对低风险项，制定应急预案，接受风险存在 |
