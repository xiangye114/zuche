# 在线租车网站系统毕业设计论文生成计划

## 一、项目概述

### 1.1 系统名称
在线租车网站系统

### 1.2 技术栈
| 层次 | 技术 | 版本 |
|------|------|------|
| 前端框架 | Vue.js | 3.4.0 |
| 前端路由 | Vue Router | 4.2.5 |
| 状态管理 | Pinia | 2.1.7 |
| UI组件库 | Element Plus | 2.4.4 |
| HTTP客户端 | Axios | 1.6.2 |
| 图表库 | ECharts | 5.4.3 |
| 后端框架 | Spring Boot | 2.7.18 |
| ORM框架 | MyBatis Plus | 3.5.5 |
| 安全框架 | Spring Security | - |
| 认证方式 | JWT | 0.12.3 |
| 数据库 | MySQL | 5.7+ |
| 缓存 | Redis | - |
| 实时通信 | WebSocket | - |

### 1.3 核心功能模块
1. **用户管理模块**：注册、登录、个人信息管理、权限控制
2. **车辆管理模块**：车辆浏览、详情查看、收藏、分类筛选
3. **订单管理模块**：订单创建、6状态流转管理、车辆检查、费用结算
4. **门店管理模块**：门店信息管理、位置服务
5. **智能客服模块**：基于知识库的智能问答系统
6. **推荐系统模块**：个性化推荐、相似车辆推荐
7. **后台管理模块**：用户管理、车辆管理、订单管理、数据统计

## 三、数据库设计详情

### 3.1 核心数据表

| 表名 | 说明 | 主要字段 |
|------|------|----------|
| users | 用户表 | id, username, password, phone, email, role, status |
| cars | 车辆表 | id, name, brand_id, category_id, store_id, price_per_day, status |
| orders | 订单表 | id, order_no, user_id, car_id, start_time, end_time, status, total_price |
| stores | 门店表 | id, name, address, phone, latitude, longitude |
| brands | 品牌表 | id, name, logo |
| categories | 分类表 | id, name, description |
| reviews | 评价表 | id, user_id, car_id, order_id, rating, content |
| favorites | 收藏表 | id, user_id, car_id |
| chat_sessions | 聊天会话表 | id, user_id, title, status |
| chat_messages | 聊天消息表 | id, session_id, role, content |
| knowledge_base | 知识库表 | id, question, answer, category, priority |
| order_logs | 订单日志表 | id, order_id, operator_id, from_status, to_status |
| order_car_checks | 车辆检查表 | id, order_id, check_type, check_item, check_result |
| cancel_reasons | 取消原因表 | id, reason_code, reason_name, reason_type |

### 3.2 订单状态设计
| 状态值 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 待确认 | 订单创建后等待管理员确认 |
| 1 | 已确认 | 管理员确认通过，等待取车 |
| 2 | 已取车 | 用户已取车，用车中 |
| 3 | 已还车 | 用户已还车，待结算 |
| 4 | 已完成 | 订单结算完成，流程结束 |
| 5 | 已取消 | 订单已取消，流程结束 |

## 四、系统创新点

### 4.1 完整的订单状态管理流程
- 实现6状态订单流转（待确认→已确认→已取车→已还车→已完成/已取消）
- 支持车辆取还车检查记录
- 完善的费用结算机制
- 订单状态变更日志追踪

### 4.2 智能客服系统
- 基于知识库的智能问答
- 支持多轮对话
- 关键词匹配算法
- 可扩展的知识库管理

### 4.3 个性化推荐系统
- 基于用户行为的推荐算法
- 考虑品牌偏好、车型偏好、价格偏好
- 相似车辆推荐
- 热门车辆推荐

### 4.4 多角色权限控制
- 超级管理员：全部权限
- 门店管理员：本门店数据管理
- 普通用户：租车服务使用
