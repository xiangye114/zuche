# 租车系统 - 订单逻辑重构产品需求文档

## Overview
- **Summary**: 重构租车系统的订单逻辑，实现支付确认弹窗、订单状态流转、30分钟未付款自动取消、门店管理员确认用车和还车功能，并更新前端订单页面的页签显示。
- **Purpose**: 优化用户租车流程，确保订单状态管理清晰可控，提升用户体验和系统可靠性。
- **Target Users**: 租车用户和门店管理员

## Goals
- 实现预定车辆时的支付确认弹窗
- 建立完整的订单状态流转：未付款 → 待取车 → 用车中 → 已还车
- 实现30分钟未付款订单自动取消功能
- 实现门店管理员确认用车和还车功能
- 更新前端订单页面，只显示全部、待取车、用车中、已还车四个页签

## Non-Goals (Out of Scope)
- 不修改现有用户认证和权限系统
- 不修改车辆管理和门店管理功能
- 不实现在线支付集成，只模拟支付流程
- 不修改数据库结构

## Background & Context
- 现有系统已实现基本的订单创建和管理功能
- 前端使用Vue.js 3 + Element Plus
- 后端使用Spring Boot
- 订单状态管理需要优化，缺少支付确认和自动取消机制

## Functional Requirements
- **FR-1**: 预定车辆时弹出支付确认弹窗，显示订单详情和倒计时
- **FR-2**: 订单创建后状态为未付款，30分钟内未付款自动取消
- **FR-3**: 支付成功后订单状态变更为待取车
- **FR-4**: 门店管理员确认用车后订单状态变更为用车中
- **FR-5**: 门店管理员确认还车后订单状态变更为已还车
- **FR-6**: 前端订单页面只显示全部（包含取消和未付款）、待取车、用车中、已还车四个页签

## Non-Functional Requirements
- **NFR-1**: 支付确认弹窗应包含30分钟倒计时，实时显示剩余时间
- **NFR-2**: 订单状态变更应实时反映在前端页面
- **NFR-3**: 系统应定期检查并取消超过30分钟未付款的订单
- **NFR-4**: 前端页面应响应式设计，适配不同设备

## Constraints
- **Technical**: 基于现有Vue.js 3和Spring Boot技术栈
- **Business**: 保持现有业务流程不变，只优化订单状态管理
- **Dependencies**: 依赖现有后端API和前端组件

## Assumptions
- 系统已实现基本的用户认证和权限控制
- 门店管理员已有相应的管理权限
- 现有数据库结构支持订单状态管理

## Acceptance Criteria

### AC-1: 支付确认弹窗
- **Given**: 用户选择车辆并点击预定按钮
- **When**: 系统弹出支付确认弹窗
- **Then**: 弹窗显示订单详情和30分钟倒计时
- **Verification**: `human-judgment`

### AC-2: 订单状态流转
- **Given**: 用户创建订单
- **When**: 系统创建订单并设置状态为未付款
- **Then**: 订单状态显示为未付款，30分钟内未付款自动取消
- **Verification**: `programmatic`

### AC-3: 支付成功
- **Given**: 用户在支付确认弹窗中点击确认支付
- **When**: 系统处理支付请求
- **Then**: 订单状态变更为待取车
- **Verification**: `programmatic`

### AC-4: 门店管理员确认用车
- **Given**: 订单状态为待取车
- **When**: 门店管理员确认用车
- **Then**: 订单状态变更为用车中
- **Verification**: `programmatic`

### AC-5: 门店管理员确认还车
- **Given**: 订单状态为用车中
- **When**: 门店管理员确认还车
- **Then**: 订单状态变更为已还车
- **Verification**: `programmatic`

### AC-6: 前端订单页签
- **Given**: 用户进入订单页面
- **When**: 系统加载订单数据
- **Then**: 页面只显示全部、待取车、用车中、已还车四个页签
- **Verification**: `human-judgment`
