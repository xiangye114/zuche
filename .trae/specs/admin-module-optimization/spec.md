# 后台管理模块优化设计规格说明

## 概述

本文档基于租车网站现有后台管理系统，结合行业主流设计理念与最佳实践，提供针对性的优化方案。优化将严格遵循以下原则：
- 深度融合现有业务逻辑和功能模块
- 优先利用现有功能组件
- 严格控制新功能开发范围
- 基于users表的role字段实现权限控制

## 现有系统分析

### 技术架构
- **前端**: Vue 3 + Element Plus + Vite + Pinia
- **后端**: Spring Boot 2.7 + MyBatis Plus + Spring Security + JWT
- **数据库**: MySQL + Redis

### 现有功能模块
1. **仪表盘** - 数据概览、统计卡片、最近订单、快捷操作
2. **车辆管理** - 车辆CRUD、筛选查询、状态管理
3. **订单管理** - 订单列表、状态流转、取车/还车确认
4. **用户管理** - 用户列表、状态管理
5. **门店管理** - 门店CRUD、状态管理
6. **品牌管理** - 品牌CRUD
7. **分类管理** - 车型分类CRUD
8. **知识库管理** - 知识条目管理

### 现有权限体系
- 基于users表的role字段
- role=0: 普通用户
- role=1: 管理员

### 现有设计不足识别

#### 1. 权限控制粒度不足
- **现状**: 仅区分普通用户和管理员
- **不足**: 无法满足多层级管理需求，门店管理员与系统管理员权限相同

#### 2. 表格交互体验欠佳
- **现状**: 基础表格展示，仅支持单条操作
- **不足**: 缺少批量操作、数据导出等效率功能

#### 3. 移动端适配不完善
- **现状**: 基础响应式布局
- **不足**: 表格在移动端显示不佳，操作困难

#### 4. 数据可视化程度低
- **现状**: 仪表盘仅展示基础统计数据
- **不足**: 缺少趋势图表，无法直观了解业务变化

#### 5. 操作审计缺失
- **现状**: 无操作日志记录
- **不足**: 敏感操作无法追溯

---

## 优化设计方案

### 一、权限管理体系优化

#### 1.1 权限等级定义（基于现有role字段）
```java
public enum UserRole {
    USER(0, "普通用户", "基础操作权限"),
    SUPER_ADMIN(1, "超级系统管理员", "系统全部操作权限"),
    STORE_ADMIN(2, "门店管理员", "门店相关管理权限");
    
    private final int code;
    private final String name;
    private final String description;
}
```

#### 1.2 权限控制矩阵

| 功能模块 | 超级管理员(role=1) | 门店管理员(role=2) | 普通用户(role=0) |
|---------|------------------|------------------|----------------|
| 仪表盘 | 全部数据 | 本门店数据 | 个人数据 |
| 车辆管理 | 全部车辆 | 本门店车辆 | 仅查看 |
| 订单管理 | 全部订单 | 本门店订单 | 个人订单 |
| 用户管理 | 全部用户 | 本门店用户 | 仅自己 |
| 门店管理 | 全部门店 | 仅自己门店 | 无权限 |
| 品牌管理 | 读写权限 | 仅查看 | 仅查看 |
| 分类管理 | 读写权限 | 仅查看 | 仅查看 |
| 知识库管理 | 读写权限 | 读写权限 | 仅查看 |

#### 1.3 实现方案
- **前端**: 基于role值动态渲染菜单和按钮
- **后端**: 基于role值进行数据范围过滤
- **数据权限**: 门店管理员只能操作本门店相关数据

---

### 二、表格功能增强（基于现有表格组件）

#### 2.1 批量操作功能
在现有表格基础上添加批量操作能力：

**用户管理**:
- 批量启用/禁用用户状态
- 批量删除用户

**车辆管理**:
- 批量修改车辆状态（可租/维修中/已租）
- 批量删除车辆

**订单管理**:
- 批量导出订单数据
- 批量状态筛选

#### 2.2 数据导出功能
基于现有查询条件，支持Excel导出：
- 导出当前筛选结果
- 支持大数据量分页导出
- 导出字段与表格列一致

#### 2.3 表格交互优化
- 列宽自适应
- 固定操作列
- 行hover效果增强
- 空数据状态优化

---

### 三、移动端适配优化

#### 3.1 布局适配
- **侧边栏**: <768px时改为抽屉式菜单
- **表格**: 小屏幕下支持横向滚动或卡片式展示
- **表单**: 移动端单列布局，增大触摸区域

#### 3.2 交互优化
- 按钮触摸区域不小于44px
- 表单输入框高度适配移动端
- 日期选择器移动端优化

---

### 四、仪表盘数据可视化

#### 4.1 基于现有数据扩展图表
利用现有Dashboard接口返回的数据，添加以下图表：

**收入趋势图**:
- 近7天/30天收入曲线
- 基于订单完成数据统计

**订单状态分布**:
- 饼图展示各状态订单占比
- 基于订单status字段统计

**车辆状态统计**:
- 柱状图展示车辆状态分布
- 基于车辆status字段统计

**用户增长趋势**:
- 近30天用户增长曲线
- 基于用户createTime统计

---

### 五、操作审计日志

#### 5.1 记录范围
- 用户状态变更操作
- 车辆信息增删改操作
- 订单状态流转操作
- 门店信息变更操作

#### 5.2 日志内容
- 操作人信息（userId, username, role）
- 操作时间
- 操作类型和模块
- 操作对象ID
- 变更前后数据（JSON格式）
- IP地址

#### 5.3 实现方式
- 使用Spring AOP切面记录
- 不改动现有业务代码
- 异步写入日志，不影响主流程性能
