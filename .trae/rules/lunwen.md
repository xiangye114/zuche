一、核心原则（贴合论文案例，强化针对性）

1. 贴合案例架构：参照“黄冈师范学院羽毛球馆在线预约系统”论文结构，围绕七大核心章节，不偏离同类论文常规框架。
2. 技术匹配精准：结合案例核心技术栈（Spring Boot、Vue等），技术描述贴合系统，不冗余。
3. 内容务实具体：参照案例风格，避免空泛，内容需有具体支撑（如预约算法、数据库设计）。
4. 学术规范统一：遵循案例文献格式、注释及章节编号，符合高校本科毕设要求。
5. 用户主导核心：AI仅辅助，核心观点、创新点、数据、代码由用户主导，不编造。

二、前期沟通与信息确认（新增案例贴合项）

1. 明确核心参数：确认高校、场馆类型、技术栈、字数及设计图需求，参照案例图表规范。
2. 对齐框架细节：结合案例，确认章节细分逻辑及增减需求。
3. 确认创新点：参照案例，引导用户明确自身创新方向，贴合系统且不重复。
4. 明确技术边界：确认用户技术基础，确定AI辅助范围（原理解释或应用描述）。

三、各章节撰写规范（结合案例优化，明确重点）

3.1 绪论（参考案例第1章）

1. 选题背景：结合场馆管理痛点及相关政策，说明开发必要性。
2. 选题意义：分实际（优化运营、改善体验）和技术（技术融合、提供参考）意义。
3. 研究现状：按案例维度总结成果、指出不足，引出研究方向。
4. 研究内容与结构：明确系统功能、技术选型，结构清晰可参考案例表格。

3.2 相关技术研究（参考案例第2章）

1. 技术分类：按案例“前端、后端、数据库、关键算法”分类，不堆砌。
2. 技术描述：说明“核心作用+系统应用”，贴合实际，不空谈原理。
3. 关键算法：简要说明逻辑及系统作用，不展开代码。

3.3 需求分析（参考案例第3章）

1. 角色与功能：参照案例双角色，明确核心功能，可辅助绘制用例图。
2. 功能需求：明确核心模块，描述具体，可参考案例表格格式。
3. 数据与可行性：明确核心数据，从经济、技术、运行环境分析可行性。

3.4 系统设计（参考案例第4章）

1. 总体架构：参照案例前后端分离架构，明确分层及作用，可辅助绘图。
2. 功能模块：细化子功能，绘制结构图，明确模块关联。
3. 数据库设计：按案例逻辑完成概念、逻辑、物理设计，表格规范。
4. 核心流程：绘制流程图，简要说明逻辑。

3.5 系统实现（参考案例第5章）

1. 开发环境：明确工具、配置及目录结构，简要说明各层作用。
2. 功能实现：按模块说明思路与效果，可搭配截图。
3. 技术细节：简要说明关键要点，需时提供代码框架。

3.6 系统测试（参考案例第6章）

1. 测试概述：明确目的、范围及核心目标。
2. 测试用例：按模块设计，覆盖核心功能，包含关键要素。
3. 结果分析：说明结论、不足及优化方向，贴合实际。

3.7 总结与展望（参考案例第7章）

1. 总结：梳理核心工作，重点说明创新点和应用价值。
2. 展望：结合实际提出合理优化方向，贴合系统需求。

3.8 参考文献与致谢（参考案例格式）

1. 参考文献：按GB/T 7714格式，优先近3-5年相关文献，数量达标。
2. 致谢：参照案例风格，涵盖相关人员，真诚规范。

2. 禁止编造内容：不编造功能、数据、文献等任何相关内容。
3. 禁止偏离框架：不擅自修改模板结构，严格参照案例框架。
